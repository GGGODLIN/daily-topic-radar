⚠️ 今日最大事件：GPT-5.5-Cyber 證明「漏洞挖掘奇點」不是 Mythos 專利——然後 OpenAI 馬上做了跟 Anthropic 一樣的事

核心矛盾：AISI（英國 AI 安全研究院）實測 GPT-5.5-Cyber 的網路安全能力，結果跟 Anthropic 的 Mythos 不相上下甚至更強；OpenAI 之前公開嘲諷 Anthropic 限制 Mythos 存取是「偽善」，現在自己也把 GPT-5.5-Cyber 鎖在「關鍵網路防禦者」專用門檻後面。

① AISI 評估：GPT-5.5-Cyber 達到甚至超越 Mythos 的網路安全水準

  • AISI 在多項挑戰中測試 GPT-5.5-Cyber——其中一項挑戰人類專家花了 12 小時才完成，GPT-5.5-Cyber 用了 11 分鐘、花費 $1.73。
  • 結論：「漏洞挖掘奇點」（vulnerability discovery singularity）不是特定模型的特徵，而是能力趨勢——只要模型規模與訓練方法到位，任何廠商都能達到類似水準。
  • 對 04-30 digest 中 DEVCORE 許復凱「奇點早在 Opus 4.6 釋出時就降臨」的判斷提供了第二份獨立驗證。
  • Anthropic Mythos 的「獨家優勢」敘事被打破——安全研究社群不再能說「只有 Anthropic 的模型有這種能力」。

② OpenAI 邊嘲諷邊複製：限制 GPT-5.5-Cyber 存取

  • TechCrunch 報導：OpenAI 此前公開批評 Anthropic 把 Mythos 限制在 Project Glasswing 計畫內是「偽善的 gatekeeping」，結果自己也宣布 GPT-5.5-Cyber 僅開放給「critical cyber defenders」使用。
  • The Register 標題一針見血：「OpenAI locks GPT-5.5-Cyber behind velvet rope despite slamming Anthropic for doing exactly that」。
  • 兩家公司面對同一個結構性困境：模型能力太強→不能隨便放出來→限制存取→被批偽善→但還是得限制。這不是道德問題，是產業規律。

③ Ars Technica：研究者確認 GPT-5.5 跟 Mythos 一樣好

  • Ars Technica 引述多位獨立安全研究者的測試結果——GPT-5.5 在自動漏洞挖掘、PoC 生成、漏洞鏈組合等面向與 Mythos 不相上下。
  • Implication：安全領域的 AI 能力護城河不存在。這對整個產業的意義比任何單一模型發布都深遠——攻擊者不需要等 Anthropic 的 model weights 洩漏，OpenAI 的模型同樣能做到。

跟你最相關：你做 daily digest 時安全是高優先主題。今天的 AISI 評估把 04-30 的 Mythos / Project Glasswing 敘事從「Anthropic 獨家威脅」升級為「產業級趨勢」——這意味著你在未來 digest 中不該只追 Anthropic 一家的動態，OpenAI、Google DeepMind、xAI 的安全模型發布都同等重要。對你個人工作流：你用的 Claude Code + Opus 4.6 已經在「漏洞挖掘奇點」這一側；今天 GPT-5.5-Cyber 的表現確認這條路是雙向的——用來防禦（紅隊測試）跟用來攻擊（自動化漏洞挖掘）是同一個能力的不同面。

🔥 Linux Copy Fail 漏洞（CVE-2026-31431）：9 年老漏洞，本機提權到 root

一個潛伏 9 年的 Linux 核心漏洞浮出水面——CVSS 7.8，影響多個主要發行版，CISA 已加入已知被利用漏洞（KEV）清單。

  • The Hacker News：Copy Fail 漏洞可讓本機使用者提權到 root，影響 Ubuntu、Debian、RHEL、SUSE 等多個主要發行版。
  • Ars Technica：「多年來最嚴重的 Linux 威脅浮現，全球緊急應對」——各大發行版同步釋出修補程式。
  • 漏洞專屬網站 copy.fail 已上線，含技術細節與 PoC。
  • Ubuntu 基礎設施在試圖就此事溝通時遭到親伊朗駭客組織 313 Team DDoS 攻擊——漏洞通報流程本身成了攻擊時機。
  • CISA 將此漏洞加入 KEV 清單，聯邦機構有期限修補。

跟你最相關：你的 launchd job 跑在 macOS 上所以不受直接影響，但如果你有 Linux server / Docker container 在跑 PullMD 或其他服務，應該確認 kernel version。313 Team 在漏洞通報期間對 Ubuntu 發動 DDoS 這件事值得記一筆——它示範了「漏洞公開時間窗口」可以被武器化為攻擊掩護。

🚨 cPanel CVE-2026-41940（CVSS 9.8）：身分驗證繞過，已遭積極利用

cPanel 爆出 CVSS 9.8 的身分驗證繞過漏洞——影響數百萬網站，已有勒索軟體受害案例出現。

  • iThome：cPanel 緊急修補 CVE-2026-41940——未驗證攻擊者可繞過身分驗證直接存取管理後台。
  • 已有第一起勒索軟體受害報告——攻擊者利用此漏洞加密伺服器資料並索取贖金。
  • CISA 將此漏洞加入 KEV 清單——從漏洞公開到第一起勒索攻擊的時間窗口極短。
  • cPanel 仍是共享主機生態的統治性基礎設施——一個漏洞就能連鎖影響數百萬站點。

觀察：CVSS 9.8 + 身分驗證繞過 + 已遭利用 + 勒索軟體——這是 2026 年目前為止對 Web 託管生態最嚴重的單一漏洞。如果你或你認識的人有網站跑在 cPanel 上，這是今天最該處理的事。

📦 供應鏈攻擊浪潮持續：PyTorch Lightning + SAP NPM + Ruby Gems + Go Modules

供應鏈攻擊沒有放緩的跡象——本日四個獨立案例覆蓋 Python、JavaScript、Ruby、Go 四大生態系。

① PyTorch Lightning PyPI 帳號被入侵用於竊取憑證

  • PyTorch Lightning 的 PyPI 發布帳號被入侵，攻擊者上傳了含 credential-stealing 後門的惡意版本。
  • 這是 PyTorch 生態一個月內第二次供應鏈事件——跟 04-30 的 LiteLLM SQL injection 事件形成 pattern。
  • ML / AI 工具鏈正在成為供應鏈攻擊的高價值目標——這些 package 常被以 pip install 一條指令安裝、不經版本鎖定。

② SAP NPM 套件遭 Mini Shai-Hulud 攻擊

  • SAP 相關的 NPM 套件被「Mini Shai-Hulud」攻擊手法入侵——利用 NPM 的信任機制注入惡意程式碼。
  • SAP 生態在企業內部的部署密度極高，這次攻擊的 blast radius 可能比表面看到的大得多。

③ 毒化 Ruby Gems 與 Go Modules 針對 CI 管線

  • 攻擊者在 Ruby Gems 和 Go Modules 中植入惡意程式碼，專門針對 CI/CD 管線——在 build 階段竊取 secrets、token、部署憑證。
  • CI 管線攻擊的危險之處在於：你信任自己的 code，也信任 CI runner，攻擊者只需要污染你信任的其中一個 dependency。

跟你最相關：你的 Python 工作流（daily-topic-radar 用 uv run）依賴 PyPI 套件。今天 PyTorch Lightning 的案例再次證明：pip install 不鎖版本等於把 supply chain trust 全押在 PyPI 帳號安全上。建議：(a) daily-topic-radar 的 requirements.txt 或 pyproject.toml 確認所有 dependency 有鎖版本；(b) CI runner 的 secrets 應該用 short-lived token 而非 long-lived API key。

🛠️ AI 開發工具安全：Cursor API Key 外洩 + Gemini CLI CVSS 10.0 RCE

開發者日常使用的 AI 工具本身成了攻擊面——兩起獨立事件覆蓋 IDE 與 CLI 兩個介面。

① Cursor IDE 設計缺陷暴露 API Key

  • LayerX 報告指出 Cursor IDE 存在設計層面的缺陷——使用者的 API key 在特定操作流程下會被暴露。
  • 這不是 bug 而是設計問題——修補需要架構層面的調整，不是簡單 hotfix。
  • 結合 04-30 PocketOS「9 秒刪庫」事件，IDE / coding agent 的安全問題正在從「偶發事件」變成「結構性風險」。

② Gemini CLI CVSS 10.0 RCE 漏洞已修補——但可能破壞 CI/CD 管線

  • Google 的 Gemini CLI 被發現 CVSS 10.0 的遠端程式碼執行（RCE）漏洞——完美分數，最高嚴重度。
  • Google 已修補，但修補方式可能破壞依賴 Gemini CLI 的 CI/CD 管線——因為修補涉及權限模型的收緊。
  • 這跟 04-30 的 OpenClaw 6 個 CVE 形成 pattern：AI agent 工具的安全基線普遍偏低，因為整個品類在過去 12 個月才爆發成長、安全審計跟不上功能開發速度。

跟你最相關：你重度使用 Claude Code（= CLI 介面）+ Cursor（= IDE 介面）。今天兩起事件精準命中你的兩個核心工具。建議：(a) 如果你用 Cursor，檢查 API key 是否在版本控制中暴露；(b) 如果你有用 Gemini CLI 跑 CI，今天修補後要測 pipeline 是否還能跑。
