⭐ v3 最大 miss + 我自己一輪誤判：Anthropic Mythos 是不公開的下一代模型

修正紀錄：本 supplement 第一版把 Mythos 描述成「Anthropic ship 的 vulnerability scanner / 工具」，是錯的。使用者指出後查 9 個 source（SecurityWeek / Engadget / Slashdot / The Mozilla Blog / TheNextWeb / Cybernews / Gigazine / TweakTown / The Register / Android Headlines）一致確認：Mythos 是 Anthropic 未公開的下一代 Claude 模型，能力強到只透過「Project Glasswing」釋出給少數 trusted partners。下方內容已重寫；The Register「magic code-sniffer」的「tool」框架是 editorial 用語、不是事實。

事件本體：Mythos（全名 Claude Mythos / Claude Mythos Preview）是 Anthropic 一個 cybersecurity-focused 的下一代模型。Anthropic CTO 對它的描述（verbatim）：

"Mythos Preview is every bit as capable as elite security researchers, and so far they've found no category or complexity of vulnerability that humans can find that this model can't."

戰績量化（vs Claude Opus 4.6）：

  • Firefox 148：Anthropic + Mozilla 合作、用 Claude Opus 4.6 跑、找到 22 個 bug（earlier this year）
  • Firefox 150：用 Claude Mythos Preview（早期版）跑、找到 271 個 bug（12 倍以上）
  • Firefox 150 公告中 40+ CVE 修補、其中 3 個（CVE-2026-6746、6757、6758）明確 credit Claude；其餘 268 個推測是低嚴重度或不達 CVE 公開門檻的 bug

釋出狀態：刻意限制公開

  • 限定計畫名稱：Project Glasswing
  • 已知夥伴：Amazon、Apple、Microsoft等 trusted technology companies（Mozilla 也是合作對象）
  • The Register 對此的評論（verbatim）："Project Glasswing, limiting early use to trusted partners with a real need, is probably a responsible approach."
  • 已發生 unauthorized access 事件：「a group of unauthorised users gained access to Mythos Preview by guessing the model's URL through a third-party vendor environment」(WebSearch 摘要原話)，Anthropic 仍在調查 — 連 Mythos 自己的 deployment 都被 Authority Gap 攻破，諷刺強到輪迴

媒體反應分裂（cross-source 對比）：

  • 正面：Engadget / SecurityWeek / Gigazine「good news for defenders」
  • 反方：The Register「271 Firefox flaws — none a human couldn't spot」(04-22) + 04-27「more Swiss cheese than cheddar」 — 論點是「Mythos 找的都是 humans 已知 vuln class，沒找到 humans 沒發現的新 class」、能力沒超越人類、只是規模放大
  • 後續：The Hacker News 04-27「Mythos Changed the Math on Vulnerability Discovery — 大多團隊還沒準備好處理修補量」 — focus 從「Mythos 強不強」轉到「下游 patch 流程跟不跟得上」

跟 v3 主 digest 直接接上：v3 在「📱 OpenAI 手機浪潮」section 引了 r/OpenAI「最強 model 都不開放給 public」(51↑、59 留言)——Mythos 就是 Anthropic 對應的代表案例。v3 把這條當「OpenAI commercial vs public 落差」，但實際上是跨家現象：OpenAI 留住 GPT-5.5/5.5 Pro 不公開、Anthropic 留住 Mythos 走 Project Glasswing，頂級模型在 2026 年正在系統性退出 public-facing tier。

v3 sit on 沒識破的另一條：v3 Reddit section 引「Pen-Testing Company XBOW on GPT-5.5: Mythos-like Cyber-Sec」(r/singularity 33↑、10 留言)——標題裡的「Mythos-like」是把 Mythos 當業界 reference benchmark 來類比（如同說「GPT-like」「Claude-like」），證明這名字在 security circle 已經是 dedicated 模型標的，v3 把它當隨機形容詞是 brand recognition fail。

跟你最相關：你 vibe coding 工作流跑 Claude Opus 4.7（1M context）— Mythos Preview 是專門 cybersec fine-tune 的同代強模型，意味著 Anthropic 認為 Opus 等 generalist 不夠專。對使用者實際影響：(a) 短期內 Claude Code / API 拿不到 Mythos 級 security 能力（Glasswing 不放）；(b) Anthropic 商業策略不是「把所有能力包進 Claude」、而是分軌釋出，跟 OpenAI 同向；(c) v3 寫的「Anthropic 商業面三重夾擊」（消費者面 friction）跟「Project Glasswing 限定」（B2B 高端只給 hyperscaler）疊起來看，是策略上的雙軌，不是矛盾。

🪤 v3 提了但細節漏了：Checkmarx 供應鏈事件升級

v3 引了 element-data Python 套件（1M monthly downloads 被竊密）作為「供應鏈攻擊」代表。這條附近其實有更大的事件 v3 沒抓到：

  • 「持續中的供應鏈攻擊『明確以 security / dev tools 為目標』」(The Register 04-27 23:33) — Lapsus$ claim、targeting 的是 security vendors 自己。
  • Checkmarx 確認 dark web 上有 GitHub repo data dump(The Hacker News 04-27 14:19) — 3/23 攻擊事件持續發酵、原始碼 + secrets 已外流。

Checkmarx 是軟體安全測試廠商本身——攻擊者選定鏈條中的 security tooling vendor 下手，就是供應鏈攻擊的 meta game。對應 v3「Cursor + Railway 災難」教訓：「destructive ops enforcement 要下沉到 token / API / infra」——但如果 token / API / infra本身的 vendor 被攻陷，downstream 全失守。Checkmarx 事件示範這條鏈條 fragility 比 v3 寫的還更深。

🐛 v3 完全沒抓到：73 個假 VS Code extension + GlassWorm v2

研究員揭露 73 個假 VS Code extension、夾帶 GlassWorm v2 malware(The Hacker News 04-27 11:23)。

跟你最相關：你在 VS Code / Cursor 跑 vibe coding，extension 是編輯器 attack surface。配 v3「Cursor 刪庫」事件，攻擊面從「agent 自主執行誤操作」延伸到「編輯器 plugin 主動帶毒」——前者是 in-context 行為失控，後者是環境 supply chain 被植入。建議檢查：(a) 你 Claude Code / Codex CLI 用的 VS Code 設定是否有非官方 extension；(b) Cursor 內 extension 來源是否限定 marketplace verified。

🤖 v3 沒抓到的 AI agent 安全主軸：Authority Gap

「Bridging the AI Agent Authority Gap: Continuous Observability as the Decision Engine」(The Hacker News 04-24)。

AI agent 安全的新框架詞彙：「Authority Gap」= agent 拿到的權限名義 vs 實際能做動作之間的落差（例如：給 agent 一個「只讀」 token，但下游 API 沒實際 enforce read-only、agent 仍能寫）。這詞 v3「Cursor + Railway」事件就是典型 case——Railway token 名義上限定 staging，實際上能打 production volumeDelete。這個詞值得固化進你 token-scope audit checklist，下次跟團隊講這個概念時用「Authority Gap」比「token scope」精確。
