🌊 最大安全事件：NCSC 警告「補丁海嘯」即將來臨

事件本體：英國國家網路安全中心（NCSC）發出罕見高層級警告——AI 驅動的漏洞獵捕工具正以前所未有的速度挖出軟體缺陷，多年累積的技術債務即將一次性爆發。原文引述："The bill for years of technical shortcuts is coming due, and it's arriving all at once."

  • NCSC 指出 AI 工具讓漏洞發現速度遠超組織修補能力，過去靠「沒人發現」存活的 legacy code 不再安全。
  • 這不是預測——Google Project Zero、DARPA AIxCC、CrowdStrike 的 AI 紅隊實測都已驗證 AI 挖洞效率是人工的數倍。
  • 來源：The Register — NCSC brace for patch tsunami

為什麼這條今天最重要：這不是單一漏洞事件，而是 NCSC 作為國家級資安機構首次把「AI 挖洞 → 技術債務一次性到期」定義為系統性風險。對你的啟示：social-info / cc-quota-fetcher 都跑在 Python + uv + brew 生態上，你依賴的每個 transitive dependency 都可能在下一波補丁海嘯中被標記為 vulnerable。今天同版面的 Trellix 源碼外洩、cPanel 勒索蔓延、CISA KEV 新增都是這波海嘯的前兆浪花。

🔓 Trellix 資安公司遭入侵——源碼外洩

資安廠商 Trellix（前身為 McAfee Enterprise + FireEye）確認遭受未授權存取，攻擊者成功取得源碼倉庫存取權限。這是一起資安公司自身被攻破的事件——當連防禦者的代碼都被偷走，攻擊面等級完全不同。

  • 源碼外洩意味著攻擊者可以 offline 分析防禦邏輯、繞過偵測規則、找出 0-day。
  • 這類事件（SolarWinds 2020、LastPass 2022、Trellix 2026）形成 pattern：資安供應鏈本身就是高價值目標。
  • 來源：The Hacker News — Trellix confirms source code breach

💀 cPanel 漏洞延燒——Sorry 勒索軟體已部署

CVE-2026-41940 的攻擊態勢持續升級：多個駭客組織同時利用此漏洞，受害伺服器已被部署 Go 語言編寫的「Sorry」勒索軟體。全球超過兩萬臺 cPanel 伺服器暴露在網際網路上。

  • 多組織同時利用同一漏洞 = 供應鏈攻擊的標準演化路徑：先 PoC 公開 → 多組織 fork → 勒索部署。
  • 台灣 iThome 追蹤報導指出 PoC 框架已出現，降低攻擊門檻。
  • 來源：iThome 台灣 — cPanel 漏洞 Sorry 勒索軟體

⚠️ CISA 將 Linux Copy Fail 加入 KEV 目錄

CVE-2026-31431（Linux kernel copy fail 漏洞）被 CISA 確認為actively exploited，正式加入 Known Exploited Vulnerabilities 目錄。聯邦機構有期限修補，民間組織也被強烈建議跟進。

  • KEV 加入 = CISA 有證據表明此漏洞在真實攻擊中被使用，不是理論風險。
  • 影響所有跑 Linux kernel 的伺服器、容器、IoT 裝置——範圍極廣。
  • 來源：The Hacker News — CISA adds actively exploited Linux root

跟你相關：你的 launchd job 跑在 macOS（BSD kernel），不直接受 Linux kernel CVE 影響。但如果你有任何 Docker container 或 remote server 跑 Linux，要確認 kernel 版本是否涵蓋此漏洞。NCSC 的「補丁海嘯」警告今天看來不誇張——光一天就有三個 major CVE 事件同時活躍。
